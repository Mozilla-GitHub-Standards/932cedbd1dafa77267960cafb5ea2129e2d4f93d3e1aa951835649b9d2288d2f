Firefox Desktop 32 User Sentiment Report

The release of Firefox 32 was a bit more dramatic than we are accustomed to, with 3 chemspills in the first few weeks of release:

  • 0.1: Fixed several crashes, a WebRTC issue, and SSL UX bug.
  • 0.2: Fixed a bug where some installations would not start up after updating Firefox.
  • 0.3: Fixed a critical zero-day security bug.

While we did release 3 chemspills, the chemspills were rolled out quickly enough to have minimal impact on user sentiment. These chemspills were unfortunate, but they do an important job of keeping users safe and providing the best experience we can.

Firefox 32 contained some awesome improvements as well, including:

  • Improved performance including crash recovery by improving the HTTP caching
  • Ability to view historical information for entries stored in the password manager
  • Easier back, forward, reload, and bookmarking through the context menu
  • And more!


This report will soon be a dashboard with live data and is based on our web API. Let us know if you have questions or comments about either project!
